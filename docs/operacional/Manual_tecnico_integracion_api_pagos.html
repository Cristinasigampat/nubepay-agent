Manual Técnico: Integración con la API de Pagos de NubePay

Área: Datos y Sistemas / Ingeniería · Versión: 1.4 · Audiencia: Equipos internos y partners tecnológicos autorizados

1. Introducción

Este manual describe cómo integrarse con la API de Pagos de NubePay (v3) para iniciar transferencias, consultar estados de transacción y recibir notificaciones vía webhooks. Está dirigido a desarrolladores internos y a partners que integran soluciones de cobro/pago con NubePay.

2. Autenticación

La API utiliza OAuth 2.0 con el flujo client_credentials. Todo consumidor de la API debe:

  1. Solicitar credenciales (client_id / client_secret) al equipo de Plataforma vía el portal de desarrolladores.
  2. Obtener un token de acceso llamando a POST /oauth/token.
  3. Incluir el token en el header Authorization: Bearer <token> en cada request.

Los tokens expiran a los 3600 segundos (1 hora). Se recomienda implementar renovación automática antes de la expiración.

3. Flujo típico de una transferencia

  1. El sistema cliente llama a POST /transfers con los datos de la cuenta origen, cuenta destino y monto.
  2. NubePay valida saldo, límites y reglas antifraude en tiempo real.
  3. Se retorna un transfer_id y un estado inicial (normalmente pendiente).
  4. El sistema cliente puede consultar el estado con GET /transfers/{transfer_id} o suscribirse a un webhook para recibir el evento transfer.completed o transfer.failed de forma asíncrona.

4. Ejemplo de request: crear transferencia

POST /v3/transfers
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
Content-Type: application/json

{
  "source_account_id": "acc_9f8d7a",
  "destination_account_id": "acc_1b2c3d",
  "amount": 250.00,
  "currency": "USD",
  "reference": "Pago factura #4521"
}

Respuesta esperada

{
  "transfer_id": "trf_7a8b9c",
  "status": "pendiente",
  "created_at": "2026-03-10T14:32:00Z"
}

5. Webhooks

Los partners pueden registrar una URL propia mediante POST /webhooks para recibir notificaciones en tiempo real. Eventos soportados:

  • transfer.completed
  • transfer.failed
  • account.created

Cada notificación incluye una firma HMAC en el header X-NubePay-Signature que debe validarse contra el webhook_secret asignado, para confirmar la autenticidad del origen.

6. Rate limits

Cada endpoint tiene un límite de solicitudes por minuto (ver tabla de referencia técnica en configuracion_endpoints_api.json). Al superar el límite, la API responde con código HTTP 429 y un header Retry-After indicando los segundos de espera recomendados.

7. Manejo de errores

Todos los errores siguen un formato estándar:

{
  "error_code": "ERR-204",
  "message": "Límite diario o mensual excedido",
  "transfer_id": "trf_7a8b9c"
}

La lista completa de códigos de error está documentada en el archivo de configuración de endpoints (configuracion_endpoints_api.json).
